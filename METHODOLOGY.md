# Methodology

## Research Design

This project follows a structured, stakeholder-driven analytical framework:

1. Define a focused civic question
2. Identify relevant datasets
3. Assess data quality
4. Perform exploratory data analysis
5. Validate findings
6. Communicate insights clearly

---

## Data Processing Steps

- Inspected dataset structure
- Generated a data dictionary
- Assessed missing values and patterns
- Evaluated temporal and geographic coverage
- Documented dataset limitations

---

## Statistical Methods

- Summary statistics (mean, median, variance)
- Frequency distributions
- Group-level comparisons
- Time-series trend analysis (if applicable)
- Correlation analysis

All calculations were performed using Pandas and validated programmatically.

---

## LLM Usage and Validation

LLMs were used to:

- Suggest exploratory questions
- Generate narrative summaries
- Identify potential stakeholder angles

Validation protocol:

- Recalculated all numerical outputs independently
- Verified consistency between narrative and statistical results
- Applied bias detection techniques to ensure neutral framing
- Documented prompt iterations and refinements

No numerical result in this project is based solely on LLM output.

---

## Limitations

- Municipal datasets may contain reporting bias
- Missing or incomplete records may affect interpretation
- Temporal gaps may distort trend analysis
- Findings should inform discussion, not replace formal policy analysis

---

## Ethical Considerations

- Only publicly available datasets were used
- No personally identifiable information was processed
- Claims are supported by data
- Uncertainty is clearly communicated

---

## Reproducibility

All code, transformations, and analytical steps are documented and version-controlled.

Raw data is stored separately from processed data to preserve original records.
